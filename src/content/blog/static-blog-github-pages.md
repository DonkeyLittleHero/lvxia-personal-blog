个人博客最重要的不是技术看起来多复杂，而是能长期更新、打开足够快、内容属于自己，并且维护成本可控。

## Astro 负责把内容提前生成

博客文章在构建阶段就会变成完整 HTML。访问者不需要等待浏览器执行一大段应用代码才能看到正文，这对阅读体验与搜索引擎都更友好。

需要互动的部分，例如搜索、主题切换和卡片倾斜，再用少量浏览器脚本实现。

## Markdown 让更新足够简单

每篇文章就是 `src/content/blog` 目录中的一个文件。顶部填写标题、简介、日期与分类，下面直接写正文。

这种方式不依赖数据库，也不需要后台管理系统。文件能进入版本控制，每一次修改都可以追溯。

## GitHub Pages 承担免费托管

代码推送到 GitHub 后，Actions 会安装依赖、构建网站并发布 `dist` 目录。日常更新只需要增加文章并推送。

如果以后需要独立域名，可以继续使用同一个项目；如果评论需求不复杂，也可以通过 Giscus 把 GitHub Discussions 变成评论区。

## 这套方案的边界

纯静态站不适合需要登录、私密内容、复杂订单或实时数据库的项目。但对个人作品、长期文章与公开项目记录来说，它的简单正是优势。

选择技术时，我更愿意问：三个月后还能不能轻松更新？这个博客给出的答案应该是“可以”。
